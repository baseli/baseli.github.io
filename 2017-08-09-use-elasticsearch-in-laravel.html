在Laravel中使用elasticsearch

要在现有的系统中进行分词，并对数据建立索引，从而达到优化搜索的目的。

几种方案

Mysql + Sphinx + Coreseek

Sphinx 建立索引配合 Coreseek 进行分词，但是在我打开Coreseek的官网的时候，已经提示找不到Dns了，不知道是什么情况。

xunsearch

在官网看了看xunsearch的说明，感觉可以完成我们项目中要求的功能，心里总想试试elasticsearch吧，说不定以后会使用的更多，所以也就不考虑这个了。

elasticsearch

elasticsearch的功能比较强大，目前中只是用到了一点，以后慢慢探索吧，这里用到了中文分词插件ik

具体实现

在Laravel 5.3后官方提供了全文搜索的解决方案Laravel Scout，会自动同步增，删，改的操作，但是官方目前只支持algolia驱动，但是在Scout上进行扩展是比较容易的，所以本文中采用了Scout + elasticsearch的方案来实现。
